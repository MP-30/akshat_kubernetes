##  READ ACCESS TO ALL THE PODS IN ALL THE NODES TO A SPECIFIC USER 

Namespace :

kubectl create ns team-a
kubectl create ns team-b


## create user 
adduser akshat


##
In Kubernetes RBAC, permissions are not tied to a namespace unless you use a Role and RoleBinding. If you want a user to have the same permission across all namespaces, you use:

ClusterRole → Defines permissions.
ClusterRoleBinding → Assigns those permissions to a user, group, or service account.


Example: Read Access to All Namespaces

nano realaccesstoallns.yml

apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRole
metadata:
  name: namespace-reader
rules:
- apiGroups: [""]
  resources:
  - pods
  - services
  - configmaps
  - namespaces
  verbs:
  - get
  - list
  - watch
---
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: namespace-reader-binding
subjects:
- kind: User
  name: akshat
  apiGroup: rbac.authorization.k8s.io
roleRef:
  kind: ClusterRole
  name: namespace-reader
  apiGroup: rbac.authorization.k8s.io



## press ctrl x -> y-> enter to come out

kubectl apply -f realaccesstoallns.yml


#### Lets check now if we are able to access the pods from akshat user

Can akshat view the pods?

kubectl auth can-i get pods --as=akshat -n team-a

o/p: yes

kubectl auth can-i get pods --as=akshat -n team-b

o/p: yes

Can akshat create pods?

kubectl auth can-i create pods --as=akshat -n team-a

o/p : no



Can Akshat delete pods?
kubectl auth can-i delete pods --as=akshat -n team-a
o/p: no


## we can even switch to akshat user and run the commands to see
but when you switch the user you will find" The connection to the server localhost:8080 was refused"

This means user akshat does not have a kubeconfig file, so kubectl doesn't know how to connect to the Kubernetes API Server.



sudo mkdir -p /home/akshat/.kube
sudo cp ~/.kube/config /home/akshat/.kube/config
sudo chown -R akshat:akshat /home/akshat/.kube


su - akshat
kubectl get pods

##########################

Built-in Read-Only Role

Kubernetes already provides a built-in ClusterRole called:view  (kubectl get clusterrole)


kubectl create clusterrolebinding akshat-view \
  --clusterrole=view \
  --user=akshat
