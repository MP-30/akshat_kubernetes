A user should be only able to view create etc pods in two namespace team-a and team-b ..and all othernamespace it should be readonly


#You can achieve this using multiple RoleBindings and one ClusterRoleBinding.



Step 1: Give Read Access Everywhere

Create a ClusterRoleBinding to the built-in view ClusterRole.



apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: user-read-all
subjects:
- kind: User
  name: akshat
  apiGroup: rbac.authorization.k8s.io
roleRef:
  kind: ClusterRole
  name: view
  apiGroup: rbac.authorization.k8s.io


kubectl get pods -A
kubectl describe pods
kubectl get svc


across all namespace


Step 2: Create a Role for Namespace Admin Activities

apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  name: pod-manager
  namespace: team-a
rules:
- apiGroups: [""]
  resources: ["pods"]
  verbs:
  - get
  - list
  - watch
  - create
  - update
  - patch
  - delete



Step 3: Bind User in team-a


apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  name: akshat-team-a
  namespace: team-a
subjects:
- kind: User
  name: akshat
  apiGroup: rbac.authorization.k8s.io
roleRef:
  kind: Role
  name: pod-manager
  apiGroup: rbac.authorization.k8s.io


Step 4: Repeat for team-b

apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  name: akshat-team-b
  namespace: team-b
subjects:
- kind: User
  name: akshat
  apiGroup: rbac.authorization.k8s.io
roleRef:
  kind: Role
  name: pod-manager
  apiGroup: rbac.authorization.k8s.io


Final result:

team-a     → Read + Create + Update + Delete Pods
team-b     → Read + Create + Update + Delete Pods
dev        → Read Only
qa         → Read Only
prod       → Read Only
